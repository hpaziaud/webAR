TP02 WebAR - Rapport complet : Exercices 1, 2 & 3

Objectif général

Développer des applications WebAR en utilisant les frameworks A-Frame et AR.js pour créer des expériences de réalité augmentée accessibles via navigateur web.

Informations générales

  • Dépôt GitHub : https://github.com/hpaziaud/webAR.git
  • Frameworks utilisés : A-Frame (version 1.4.0) + AR.js via CDN
  • Technologies : HTML5, WebGL, WebRTC, Three.js

Exercice 1 : Scène 3D avec A-Frame

Objectif

Créer une scène 3D interactive en utilisant le framework A-Frame pour le développement WebAR.

Fichier

index.html - Scène 3D A-Frame

Éléments de la scène 3D

1. Trois boîtes colorées

Boîte bleue (#4CC3D9) Position : (-2, 2, 0)
Boîte rouge (#EF2D5E) Position : (0, 2, 0)
Boîte jaune (#FFC65D) Position : (2, 2, 0)

Toutes les boîtes lévitent à 2 unités au-dessus du plan de base.

2. Plan de base

  • Surface verte (#7BC8A4) de 10x10 unités
  • Positionné à (0, 0, 0) avec rotation -90° sur l'axe X
  • Sert de sol pour la scène

3. Caméra

  • Position : (0, 10, 15)
  • Hauteur : 10 unités
  • Distance : 15 unités du centre
  • Orientation : Vers le centre du plan pour une vue d'ensemble

4. Éclairage

  • Éclairage ambiant : Lumière douce générale
  • Éclairage directionnel : Lumière principale pour les ombres

Exercice 2 : Intégration AR.js avec marqueur Hiro

Objectif

Intégrer AR.js et tester la réalité augmentée sur mobile avec le marqueur Hiro.

Fichier

exercice2.html - Application AR.js

Fonctionnalités AR

  • Bibliothèque AR.js : Intégrée via CDN pour la détection de marqueurs
  • Marqueur Hiro : Marqueur prédéfini pour la détection AR
  • Boîte semi-transparente : Apparaît lors de la détection du marqueur
  • Animation de rotation : La boîte tourne automatiquement sur elle-même
  • Interface mobile : Optimisée pour les smartphones

Configuration technique

Détection Mode mono_and_matrix
Marqueur Hiro (3x3 matrix)
Animation Rotation 360° en 2 secondes

Instructions d'utilisation

  1. Ouvrir exercice2.html sur un smartphone
  2. Autoriser l'accès à la caméra
  3. Télécharger le marqueur Hiro
  4. Pointer la caméra vers le marqueur
  5. Observer la boîte bleue semi-transparente qui apparaît et tourne

Technologies utilisées

  • A-Frame : Framework HTML pour la réalité virtuelle/augmentée
  • WebGL : Rendu 3D via Three.js (intégré dans A-Frame)
  • HTML5 : Structure de base

Résultat

Une scène 3D fonctionnelle accessible via navigateur web, avec trois objets colorés lévitant au-dessus d'un plan, vue depuis une caméra positionnée pour offrir une perspective optimale de la scène.

Conclusion

Démontre la simplicité d'utilisation d'A-Frame pour créer rapidement des expériences 3D web. Le framework permet de développer des scènes interactives avec une syntaxe HTML intuitive.

Exercice 3 : Interactions par clic/tap

Objectif

Ajouter des interactions par clic/tap à l'application AR, permettant à l'utilisateur d'interagir avec les objets 3D affichés via le marqueur Hiro.

Fichier

exercice3.html - Application AR.js interactive

Fonctionnalités interactives

  • Curseur de caméra : Curseur visible pour pointer et interagir avec les objets
  • Changement de couleur : La boîte change de couleur à chaque clic/tap
  • Animation de rotation variable : La vitesse de rotation change lors de l'interaction
  • Changement d'échelle : La taille de la boîte s'agrandit/réduit au clic
  • Animation de pulsation : Effet visuel lors du clic pour feedback utilisateur
  • Support multi-plateforme : Fonctionne sur desktop (clic) et mobile (tap)

Configuration technique

Curseur Ring avec animation de fusion
Interactions Click (desktop) + Touch (mobile)
Couleurs 6 couleurs cycliques
Animations Rotation (2s/0.5s) + Échelle (1x/1.5x)

Instructions d'utilisation

  1. Ouvrir exercice3.html sur un smartphone ou desktop
  2. Autoriser l'accès à la caméra
  3. Télécharger le marqueur Hiro
  4. Pointer la caméra vers le marqueur
  5. Observer la boîte bleue qui apparaît et tourne
  6. Cliquer/taper sur la boîte pour voir les changements :
  7. • Couleur change (6 couleurs disponibles)
  8. • Vitesse de rotation alterne (lente/rapide)
  9. • Taille alterne (normale/agrandie)
  10. • Animation de pulsation au moment du clic

Technologies utilisées

  • A-Frame : Framework HTML pour la réalité virtuelle/augmentée
  • AR.js : Bibliothèque JavaScript pour la réalité augmentée web
  • WebGL : Rendu 3D via Three.js (intégré dans A-Frame)
  • WebRTC : Accès à la caméra pour la détection AR
  • JavaScript : Gestion des événements et interactions
  • HTML5 : Structure de base

Résultat

Une application de réalité augmentée interactive fonctionnelle sur mobile et desktop, utilisant la détection de marqueurs pour afficher des objets 3D avec lesquels l'utilisateur peut interagir. Les interactions par clic/tap offrent un feedback visuel riche avec changements de couleur, d'animation et d'échelle.

Conclusion

Démontre l'ajout d'interactivité avancée aux applications WebAR. L'intégration d'événements de clic/tap avec des animations dynamiques crée une expérience utilisateur engageante et responsive, ouvrant la voie à des applications AR plus complexes et interactives.

Bilan global

Ces exercices facilitent l'apprentissage du développement WebAR en combinant la puissance d'A-Frame avec les capacités AR d'AR.js, ouvrant la voie à des applications de réalité augmentée web avancées et interactives.
